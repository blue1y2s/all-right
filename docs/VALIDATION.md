## 已覆盖测试

- 四个平台 adapter 均能从同一源内容生成标题、正文、摘要、标签、字数和 warnings。
- 四个平台输出风格差异可验证：公众号导语、知乎结论先行、B站看点、小红书短句场景。
- 模拟发布支持成功和失败分支。
- 发布队列支持 pending、running、success、failed、withdrawn 状态。
- 发布历史支持去重和 16 条上限。
- 草稿包支持复制文本和 JSON 导出。

## 手动验收流程

1. 启动本地服务：`npm run dev`。
2. 打开本地页面，确认四个平台默认选中。
3. 修改标题、正文、标签，确认预览实时变化。
4. 点击热点选题，确认大纲能写入正文。
5. 点击复制和导出，确认草稿流转可用。
6. 点击“一键模拟发布”，确认队列按平台流转并写入历史。
7. 对成功任务点击“模拟撤回”，确认状态变为“已撤回”并追加日志。
8. 切到“扩展架构”，确认新增平台和真实发布接入路径可解释。

## 交付边界

首版默认模拟发布。真实发布建议通过官方 API、浏览器自动化或 Wechatsync CLI/MCP 接入，并由平台 adapter 声明是否支持发布后编辑、撤回或删除。对于不支持撤回的平台，界面应提示用户到平台侧手动处理。
